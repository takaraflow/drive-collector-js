## 2024-05-18 - DriveRepository findAll Optimization\n**Learning:** The `DriveRepository.findAll` function used a sequential `for...of` loop to hydrate active drive IDs sequentially via `findById`. For users with multiple drives, this causes an N+1 latency bottleneck as it waits for each cache/DB lookup before moving to the next. Previous implementation attempted to use an unmaintainable unbounded worker pool pattern.\n**Action:** Replaced the `for...of` loop with `await Promise.all(activeIds.map(...))`. This executes all `findById` requests concurrently, drastically reducing fetch times without introducing arbitrary bottleneck limits, while maintaining clean and readable code.
